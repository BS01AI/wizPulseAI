## 📁 目录结构

```
technical-docs/
├── 00-shared/          # 共享组件和通用设计

├── 02-dashboard-site/  # Dashboard站点技术文档

└── 99-archive/         # 归档的旧文档
```

## 📝 文档命名规范

### 文件命名格式
```
{编号}-{类型}-{名称}-{版本}.md

示例：
01-ARCH-技术架构-v1.0.md
02-API-接口设计-v2.1.md
03-DB-数据库结构-v1.0.md
```

### 文档类型编码
- **ARCH** - 架构设计文档
- **API** - 接口设计文档  
- **DB** - 数据库设计文档
- **FLOW** - 业务流程文档
- **SEC** - 安全设计文档
- **DEPLOY** - 部署配置文档
- **GUIDE** - 开发指南文档
- **TEST** - 测试方案文档

### 版本号规则
- **v1.0** - 首次正式版本
- **v1.1** - 小更新（修复、优化）
- **v2.0** - 大更新（架构变更）
- **-draft** - 草稿版本
- **-review** - 审核中版本

## 🔄 文档更新流程

1. **创建草稿**: `{编号}-{类型}-{名称}-draft.md`
2. **提交审核**: `{编号}-{类型}-{名称}-review.md`
3. **正式发布**: `{编号}-{类型}-{名称}-v1.0.md`
4. **归档旧版**: 移动到 `99-archive/` 目录
