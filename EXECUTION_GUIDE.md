# 执行顺序指南 - SEO/AEO/GEO 实施策略

> **实战执行手册** | 解答"同时做还是分开做"的问题

---

## 🎯 核心策略：分层并行推进

### 优先级金字塔

```
         🏆 最终目标
        被AI引用 + 流量增长
              ↑
      ┌───────┴───────┐
      │   GEO (持续)   │  ← Layer 3: 外部引用（贯穿始终）
      └───────┬───────┘
  ┌───────────┴───────────┐
  │   AEO (内容优化)       │  ← Layer 2: 知识中心改造
  └───────────┬───────────┘
┌─────────────┴─────────────┐
│   基础SEO (技术修复)       │  ← Layer 1: 必须先做
└───────────────────────────┘
```

**关键原则**：
> **不是串行**（做完A→再做B→再做C）
>
> **而是波浪式并行**（每周有主次，逐层叠加）

## Week 1: 技术基础层（地基）⚡ 阻塞性任务

### 🎯 目标
让 Google 能抓取和索引你的网站

### 📅 Day 1-2: Schema.org 标记

**负责人**：开发工程师
**工时**：6小时

**必须完成**：
```typescript
// 1. 首页 Organization Schema
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "wizPulseAI",
  "url": "https://www.wizpulseai.com"
}

// 2. 产品页 SoftwareApplication Schema
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication",
  "name": "QuickSlide"
}

// 3. 知识中心 Article Schema
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "..."
}
```

**为什么必须先做**：
- ❌ 没有 Schema → AI 无法理解页面内容
- ❌ 后续所有 AEO/GEO 工作都依赖这个

---

```typescript
    title: '...',

    openGraph: { ... }
  };
}
```

**5个核心页面**：
1. 首页
2. QuickSlide 产品页
3. 知识中心主页
4. About
5. Contact

### 📊 为什么知识中心优先于主页？

**数据对比**：

| 场景 | 访问量 | 转化率 | 注册数 |
|------|--------|--------|--------|
| **场景A：先优化主页** | 1,000 | 1%→5% | 10→50 (**+40**) |
| **场景B：先做AEO引流** | 1,000→5,000 | 2% | 10→100 (**+90**) |

**结论**：
> 流量优先于转化
>
> 主页优化 = 加宽水管出口
> AEO/GEO = 增加水源
> **水源不够，管子再粗也没用**

## 🎯 三大核心原则

### 1️⃣ 技术债务必须先还

**类比**：
> SEO基础 = 房子地基
>
> 地基没打好，楼层再漂亮也没用

**如果跳过 Week 1**：
- ❌ 写再多内容，Google 也抓取不到
- ❌ 改造再好的文章，也不会出现 Featured Snippet
- ❌ 外部链接再多，页面打开是 404

---

### 2️⃣ 流量优先于转化

**优化顺序**：
```
❌ 错误：主页转化率 1%→5% → 然后引流
✅ 正确：先引流 1K→5K → 然后优化转化
```

**原因**：
- 小流量 × 高转化 = 小收益
- **大流量 × 中转化 = 大收益**

---

### 3️⃣ GEO 需要时间发酵

**外部引用的累积效应**：
```
Week 1: 5 个回答 → AI 不认识你
Week 2: 10 个回答 → AI 开始注意
Week 3: 15 个回答 + 媒体 → AI 可能引用
Week 4: 测试引用率 5-10%
```

**所以**：
- ✅ Week 2 就开始（持续积累）
- ❌ Week 4 才做（来不及）

## 📊 实战FAQ

### Q1: 为什么不能先优化主页再引流？

**A**: 因为主页转化优化需要数据

**没有流量时**：
- 不知道哪个 CTA 文案更好
- 不知道用户在哪里跳出
- 优化方向全靠猜

**有流量后**：
- 看数据：80% 用户在首屏跳出 → 精简首屏
- 看数据：CTA点击率 0.5% → 优化文案
- **数据驱动优化 > 盲目优化**

---

### Q2: 能否跳过 Week 1，直接做 AEO？

**A**: 不能，技术基础是阻塞性任务

**如果跳过**：
```
你写的AEO内容再好
     ↓
Google 抓不到（没sitemap）
     ↓
不会出现 Featured Snippet
     ↓
白做
```

**类比**：
> 开餐厅，菜做得再好，门没开，客人进不来

---

### Q3: Week 2 能否同时优化主页？

**A**: 可以，但不推荐

**资源分配**：
- Week 2 只有 100% 精力
- AEO 需要 80%（改造6篇文章很费时）
- GEO 需要 20%（开始积累）

**如果加主页**：
- AEO 只能做 50% → 3篇文章（不够）
- GEO 被压缩 → 外部引用不足
- 主页优化 30% → 没数据支撑，效果差

**结论**：
> 聚焦比分散更有效

---

### Q4: GEO 为什么从 Week 2 就开始？

**A**: 因为外部引用是累积过程

**AI 如何"记住"你**：
```
见到1次提及 → 不记得
见到5次提及 → 有印象
见到20次提及 → 开始引用
见到50次提及 → 默认推荐
```

**时间线**：
- Week 2 开始：20次提及需要 2-3 周
- Week 4 开始：来不及

---

### Q5: 如果 Week 1 延期怎么办？

**A**: 立即增加资源或砍掉非核心功能

**优先级排序**：
```
P0（必须做）:
  ├─ Schema 标记（首页 + 产品页）
  ├─ sitemap.xml
  └─ 3 篇文章详情页

P1（重要但可延后）:
  ├─ 全部页面 Schema
  └─ 性能优化

P2（Week 2 做也行）:
  └─ Open Graph 优化
```

**应对**：
- 只做 P0，Week 1 按时完成
- P1/P2 移到 Week 2
